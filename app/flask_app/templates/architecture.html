Engineering Philosophy

The architecture follows atmospheric dependency rather than isolated prediction.

Dependency-Aware Forecasting

Weather variables do not exist independently. Temperature influences pressure, pressure influences moisture, moisture influences rainfall, and rainfall influences sky conditions. The system models these atmospheric dependencies as a structured hierarchy instead of treating each variable as an isolated target.

Storage Responsibilities

  • Cities stores monitored locations.
  • Historical Weather stores approximately fifteen years of observations.
  • NWP Cache stores external numerical weather prediction.
  • Predictions stores generated forecasts.
  • Prediction Errors stores evaluation metrics.

Why PostgreSQL

PostgreSQL was selected for ACID compliance, reliable relational storage, efficient indexing, strong Python ecosystem support and a direct path toward future scalability.

Execution Design

The automation layer is intentionally modular. Each responsibility is isolated into an independent script. The master pipeline coordinates execution, captures failures, logs output and stops immediately if any critical stage fails.

Current Limitations

Known scope boundaries accepted for production robustness.

Training Dataset

Five monitored cities.

Temporal Resolution

Daily forecasting.

External Dependencies

Wind, cloud cover and rainfall amount supplied through Numerical Weather Prediction.

Geographical Coverage

Currently optimised for selected Indian cities.

These limitations were accepted deliberately to maximise production robustness and provide a strong foundation for future expansion.

Every subsystem is independently maintainable while contributing to a unified forecasting pipeline.
