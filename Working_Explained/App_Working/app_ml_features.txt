Overview of the Feature Pipeline

This Python code defines a class named FeaturePipeline that prepares and organizes data (features) for a multi-step
machine learning weather forecasting model. It uses the pandas and numpy libraries to extract temporal patterns,
retrieve historical database records, and combine them with intermediate model predictions to create structured
datasets for training or inference.

Here is the detailed, step-by-step breakdown of how the code works:

1. Class Initialization

Method: **init**(self, db)

Action: When a new FeaturePipeline object is created, it accepts a database connection object (db) and stores it.
This database object is used later to query historical weather data and city coordinates.

2. Time and Date Transformations

Method: _make_temporal(self, target_date)

Action: This private helper method takes a target date and extracts specific time-based numerical features.

Specific Extractions: It extracts the raw year and the day of the week.

Cyclical Transformations: Because months (1-12) and days of the year (1-365) operate in a loop (e.g.,
December 31st is close to January 1st), it converts them into continuous mathematical signals using Sine and Cosine
functions (month_sin, month_cos, day_sin, day_cos). This helps the machine learning model understand the cyclical
nature of seasons.

3. Database Data Retrieval

Method: _city(self, city_id)

Action: Retrieves static geographic data (like latitude and longitude) for a specific city from the database.

Method: _yesterday(self, city_id, target_date)

Action: Subtracts exactly one day from the target date to find "yesterday," then queries the database for the actual
recorded weather data for that city on that specific past day.

4. Rolling Averages

Method: _roll3(self, city_id, target_date)

Action: Looks at the last three days of historical data leading up to the target date.

Calculation: It calculates the average (mean) of the maximum temperatures (temp_max_c) and
minimum temperatures (temp_min_c) over that 3-day window. This provides the model with a short-term trend rather
than just a single day's temperature.

5. Building Foundational Features (Tier 1)

Method: build_tier1_features(self, city_id, target_date)

Action: This method acts as the first major assembly line. It calls all the helper methods described above to build a
foundational dataset for the given city and date.

Data Included: It combines geographic data (latitude/longitude), the cyclical time data, yesterday's radiation and
daylight duration, yesterday's max/min temperatures, and the 3-day rolling average temperatures.

Output: It packages all these data points into a single-row Pandas DataFrame, which is typically fed into the first
level of machine learning models (e.g., to predict today's temperature and pressure).

6. Assembling Rain Prediction Features

Method: assemble_rain_features(tier1, temp_pred, pressure_pred, moisture_pred, nwp)

Action: This is a static method (it does not require the class to be instantiated or use the database).
It takes the foundational "Tier 1" features and merges them with new predictions generated by earlier models in the
pipeline.

Data Mapping: It extracts the first elements of temp_pred and pressure_pred as the max values, and the
second elements as the min values. It extracts dew point max and min from moisture_pred. It pulls wind speed, wind
direction (in Sine/Cosine format), and cloud cover from an external Numerical Weather Prediction (nwp) dictionary.

Output: It returns a new Pandas DataFrame. This specific combination of features is optimized to be fed into a model
that predicts whether it will rain and how much.

7. Assembling Final Weather Code Features

Method: assemble_weathercode_features(rain_features, moisture_pred, rain_amount)

Action: This is the final static assembly step. It prepares the data needed to predict the ultimate "Weather Code"
(e.g., Sunny, Cloudy, Thunderstorm).

Data Mapping: It starts by taking an exact copy of the DataFrame generated in the Rain Features step.
It then appends two critical new pieces of information: the maximum relative humidity (extracted from the third element
of moisture_pred) and the newly predicted rain_amount.

Output: It returns the fully assembled Pandas DataFrame containing geography, time, past trends,
temperature predictions, moisture predictions, wind forecasts, and rain predictions, ready for the final classification
model.
